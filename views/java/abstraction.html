추상화?(Abstraction)

  • 세상에 있는 모든 것을 클래스를 만들고 그 클래스의 상태와 행동을 부여한 것으로 객체를 만든다.
  • 따라서 추상적으로 클래스를 구현하고 거기에 공통적인 요소나 필수적인 요소를 이용하여 만든 것이 추상클래스입니다.
  • 이 과정에서 공통적인 요소나 특징을 추출하는 과정이 추상화입니다.

    • ex) 여러 기업의 여러 자동차가 있다. 그 모든 자동차를 클래스로 만들고 모든 차에는 시동을 걸어야 간다.

          이러한 것들이 공통적인 요소나 특징이다.

    추상화의 장점?

  • 재사용성이 높다
      • 모델링을 통해 코드를 추상화하면 코드의 재사용성을 높일 수 있다.
  • 가독성 좋다
      • 코드를 추상화하였기 때문에 코드가 간결해지고 가독성이 올라간다.
  • 생산성이 증가된다.
      • 재사용성이 높고 가독성이 좋아 코드에 대한 이해가 빠르기 때문에 오류가 감소하고
      유지 보수시 시간이 단축되어 생산성이 증가한다.

    추상화의 단점?

  • 수정이 어렵다.
      • 내부에 어떠한 코드가 있는지 모르기 때문에 수정해서 사용하고 싶어도 사용하기 힘들다.

    캡슐화란?(Encapsulation)

  • 캡슐화 Encapsulation란 객체의 데이터(필드), 동작(메소드)을 하나로 묶고 실제 구현 내용을 외부에 감추는 것을 말한다.

    • 캡슐화의 특징?

  • 외부 객체는 객체 내부의 구조를 알지 못하며 객체가 노출해서 제공하는 필드와 메소드만 이용할 수 있습니다.

    • 캡슐화의 장점?

  • 코드의 안정성을 높일 수 있다.
      • 캡슐화를 통해 외부에서 객체의 상태를 직접 접근할 수 없기 때문에 객체의 내부 상태가 변하지 않고 유지된다. 이로 인해 예기치 않은 상태 변경을 방지할 수 있으며, 코드의 안정성을 높일 수 있다.
  • 객체의 내부 구현을 숨길 수 있다.
      • 캡슐화를 통해 객체의 내부 구현이 외부에 노출되지 않기 때문에 객체를 사용하는 다른 코드에 대한 의존성을 줄일 수 있다. 이로 인해 코드의 유지보수성을 높일 수 있다.
  • 코드의 재사용성을 높일 수 있다.
      • 캡슐화를 통해 객체의 내부 구현이 외부에 노출되지 않기 때문에 객체를 다른 코드에서도 쉽게 사용할 수 있다. 이로 인해 코드의 재사용성을 높일 수 있다.

    캡슐화의 단점?

  • 코드의 이해도가 떨어질 수 있다.
      • 캡슐화를 지나치게 하면 객체의 내부 구현을 숨기기 때문에 객체의 내부 동작을 이해하는 데 어려움을 겪을 수 있다.
  • 객체의 상태와 동작을 접근하기 위해 추가적인 인터페이스를 작성해야 할 수도 있다.
      • 이로 인해 코드의 복잡도가 높아질 수 있다.
