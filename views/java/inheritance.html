상속이란?(Inheritance)

  • 상속은 새로운 클래스를 정의할 때 이미 존재하는 클래스의 속성과 메서드를 물려받아 사용할 수 있게 해주는 기능이다.
    이 과정에서 기존 클래스를 '부모 클래스'라고 하며, 상속받는 새로운 클래스를 '자식 클래스'라고 한다. 자식 클래스는 부모 클래스의 특성을 상속받아, 추가적인 기능을 구현하거나 기존의 기능을 수정하여 확장된 기능을 가질 수 있다.

    • 상속의 종류?

  • 클래스 상속
      • 클래스를 확장해준다.
      extends 키워드를 사용한다.
      다중 상속이 불가능하다.
  • 인터페이스 상속
      • 인터페이스를 구현해준다.
      implements 키워드를 사용한다
      다중 상속이 가능하다.
      설계 목적으로 구현이 가능하다.

    상속의 장점?

  • 코드의 확장성, 재사용성 상승
  • 중복된 코드 제거가능

    • 상속의 단점?

  • 캡슐화를 깨트린다
      • 상위 클래스의 변경이 하위 클래스에 예기치 않은 영향을 미칠 수 있다.
      하위 클래스가 상위 클래스의 protected 멤버에 접근할 수 있어, 내부 구조에 간섭할 수 있다.
      하위 클래스가 상위 클래스의 메서드를 오버라이드하여 원래의 행동을 변경할 수 있다.
  • 부모 클래스의 필드와 메서드를 찾는데 불편함이 있다
      • 주로 상속 계층이 깊어질 때나 다중 상속이 이루어질 때 발생한다
  • 상속은 결함 역시도 상속한다
